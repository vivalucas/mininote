# MiniNote

macOS용 경량 텍스트 편집기. Windows 11 메모장 경험을 목표로 합니다.

[中文](README.md) | [English](README.en.md) | [日本語](README.ja.md) | [Deutsch](README.de.md) | [Français](README.fr.md) | [Русский](README.ru.md)

## 왜 만들었나

Windows에서 Mac으로 전환했고 대부분의 것에 익숙해졌지만, Windows 11 메모장이 계속 그리웠습니다.

강력해서가 아니라 오히려 단순하기 때문입니다. 열면 쓸 수 있고, 닫아도 사라지지 않고, 붙여넣으면 깔끔한 텍스트가 됩니다. 서식 없이, 리치 텍스트 없이, "스마트" 제안 없이. 그저 조용히 글을 쓸 수 있는 공간.

macOS에서 오랫동안 찾았습니다. 너무 무거운 것(Obsidian, Typora), 너무 단순한 것(기본 텍스트 편집기), 구독이 필요한 것. 딱 맞는 것이 없었습니다.

그래서 직접 만들었습니다.

## 기능

- **탭 영속성**
  - 새 탭을 만들면 바로 쓸 수 있고, 내용이 실시간 자동 저장
  - 종료, 재시작, 정전에도 모든 내용이 복원. 아무것도 잃지 않음
  - 창을 닫으면 모든 탭이 조용히 유지. 알림 없음
  - 저장되지 않은 변경사항이 있는 탭을 닫으면 저장 / 저장 안 함 / 취소 표시

- **이중 저장 로직**
  - Session 레이어: 실시간으로 모든 것을 기록. 재시작해도 유지
  - 디스크 레이어: 사용자가 명시적으로 Cmd+S를 누를 때만 파일에 기록
  - 두 레이어는 독립적으로 동작

- **일반 텍스트 붙여넣기**
  - 기본적으로 붙여넣기는 일반 텍스트. 추가 작업 불필요
  - 웹페이지, WeChat, PDF에서 복사한 텍스트는 서식이 자동 제거
  - "서식 청소 스테이션"으로 활용: 리치 텍스트를 붙여넣고 → 다시 복사하면 깔끔한 텍스트

- **Markdown 렌더링 (선택 사항)**
  - 기본은 일반 텍스트 편집. Cmd+R로 렌더링 뷰 전환
  - 렌더링 뷰에서는 편집 불가. 일반 텍스트로 돌아가면 편집 재개
  - .mint / .txt / .md 각각 설정에서 렌더링 ON/OFF 독립 제어

- **Finder 연동**
  - Finder의 폴더를 우클릭하여 .mint 파일 생성
  - macOS 네이티브 Quick Look 지원. 파일 선택 후 스페이스키로 미리보기
  - .mint 파일은 기본적으로 MiniNote로 열림

- **기타**
  - 상태 표시줄에 행/열, 글자 수, 인코딩, 줄 바꿈, 렌더링 모드 표시
  - 3가지 파일 형식: .mint(기본), .txt, .md. 다른 이름으로 저장으로 변환
  - 테마 전환: 라이트 / 다크 / 시스템 따름
  - 설정에서 GitHub 업데이트 확인

## 단축키

| 기능 | 단축키 |
|------|--------|
| 새로 만들기 | `Cmd+N` |
| 열기 | `Cmd+O` |
| 저장 | `Cmd+S` |
| 다른 이름으로 저장 | `Cmd+Shift+S` |
| 탭 닫기 | `Cmd+W` |
| 실행 취소 / 다시 실행 | `Cmd+Z` / `Cmd+Shift+Z` |
| 찾기 | `Cmd+F` |
| 찾기 및 바꾸기 | `Cmd+Option+F` |
| Markdown 전환 | `Cmd+R` |
| 환경 설정 | `Cmd+,` |

## 지원 형식

| 형식 | 설명 |
|------|------|
| .mint | MiniNote 고유 형식. 새 파일의 기본값. 일반 텍스트 + 경량 상태 정보 (커서 위치, 렌더링 상태) |
| .txt | 표준 일반 텍스트. 다른 편집기와 호환 |
| .md | Markdown 형식 |

세 가지 모두 본질적으로 일반 텍스트. 형식 변환은 확장자만 변경하면 됩니다.

## 시스템 요구 사항

- macOS 26 (Tahoe) 이상
- Apple Silicon Mac (M 시리즈 칩)

## 설치

**방법 1: DMG 설치**

1. [Releases](../../releases) 페이지에서 최신 `MiniNote-[버전].dmg` 다운로드
2. DMG를 열고 MiniNote를 Applications 폴더로 드래그
3. 첫 실행 시 macOS가 "앱이 손상됨" 또는 "개발자를 확인할 수 없음"을 표시할 수 있습니다. 이는 서명되지 않은 앱에 대한 Gatekeeper의 정상 동작입니다. 터미널에서 다음을 실행하여 격리 플래그를 제거:
   ```bash
   xattr -cr /Applications/MiniNote.app
   ```
   그런 다음 더블클릭하여 실행. 또는 우클릭 → 열기 → 대화상자에서 "열기" 클릭.

**방법 2: ZIP 아카이브**

1. [Releases](../../releases) 페이지에서 최신 `MiniNote-[버전].zip` 다운로드 후 압축 해제
2. `MiniNote.app`을 Applications 폴더로 이동
3. 터미널에서 실행:
   ```bash
   xattr -cr /Applications/MiniNote.app
   ```

**방법 3: 소스에서 빌드 (서명 우회 불필요)**

1. 이 저장소를 클론
2. Xcode에서 `MiniNote.xcodeproj` 열기
3. **Signing & Capabilities**에서 개발자 계정 선택
4. `Cmd+R`로 실행. Xcode가 서명을 자동 처리

## 사용법

- **새 탭**: `Cmd+N`으로 스크래치 문서 생성. 바로 작성 시작 가능
- **파일 열기**: `Cmd+O`로 디스크의 .mint / .txt / .md 파일 열기
- **저장**: `Cmd+S`로 현재 탭을 디스크에 저장. 스크래치 문서는 다른 이름으로 저장 대화상자 표시
- **다른 이름으로 저장**: `Cmd+Shift+S`로 다른 형식으로 저장 (.mint / .txt / .md)
- **렌더링 전환**: `Cmd+R`로 일반 텍스트 편집과 Markdown 렌더링 뷰 전환
- **찾기 및 바꾸기**: `Cmd+F`로 찾기, `Cmd+Option+F`로 찾기 및 바꾸기
- **탭 정렬**: 탭을 드래그하여 순서 변경
- **탭 닫기**: `Cmd+W`. 저장되지 않은 변경사항이 있으면 저장 확인 표시
- **업데이트 확인**: 환경 설정(`Cmd+,`)의 "업데이트 확인" 버튼

## 자주 묻는 질문

**스크래치 문서는 어디에 저장되나요?**

`~/Library/Application Support/MiniNote/sessions/` 디렉토리에 저장됩니다. 각 스크래치 문서는 개별 파일로 저장되며, `session.json`에 탭 순서와 메타 정보가 기록됩니다.

**.mint와 .txt의 차이는?**

본질적으로 동일. 둘 다 일반 텍스트. 유일한 차이는 .mint가 커서 위치와 렌더링 상태를 추가 저장한다는 것. 다른 이름으로 저장으로 자유롭게 변환 가능하며 내용은 변경되지 않습니다.

**구문 강조를 지원하나요?**

아니요. MiniNote는 일반 텍스트 편집기입니다. Markdown 렌더링은 시스템의 AttributedString을 사용하여 기본 제목, 목록, 굵게 등을 표시합니다. 코드 구문 강조는 지원하지 않습니다.

**TextEdit / CotEditor / BBEdit와 어떻게 다른가요?**

MiniNote의 핵심 철학은: 탭 영속성 (재시작해도 유지) + 이중 저장 로직 (스크래치와 디스크 분리). TextEdit는 탭 영속성을 지원하지 않습니다. CotEditor는 더 많은 기능을 제공하지만 이 메커니즘이 없습니다. BBEdit는 너무 무겁습니다. MiniNote는 이 하나를 잘 수행합니다.

**클라우드 동기화를 지원하나요?**

아니요. 앞으로도 지원하지 않을 것입니다. 모든 데이터는 로컬에 저장되며 완전히 오프라인입니다.

## 개발

기술 스택: Swift 6 + SwiftUI + NSTextView (TextKit). 서드파티 의존성 없음.

```bash
git clone https://github.com/vivalucas/mininote.git
open MiniNote.xcodeproj
# Cmd+B로 빌드
```
