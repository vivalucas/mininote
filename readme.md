# MiniNote

一个 macOS 上的轻量级纯文本编辑器，体验对标 Windows 11 新版记事本。

## 为什么做这个

我从 Windows 换到 Mac，很多东西都适应了，但一直想念 Windows 11 新版记事本。

不是因为它多强大，恰恰是因为它够简单。打开就写，关了不丢，粘贴过来就是干净文字。没有格式，没有富文本，没有"智能"提示。就是一个安静的写字的地方。

macOS 上找了很久，要么太重（Obsidian、Typora），要么太简陋（系统自带的文本编辑），要么要订阅。没有一个刚好够用的。

所以自己做了一个。

## 功能

- **标签页持久化**
  - 随手新建标签页即可写，内容实时暂存
  - 关机、重启、断电，内容全部恢复，什么都不丢
  - 关闭整个窗口静默保留所有标签页，不弹任何提示
  - 关闭单个标签页时如有未保存改动，弹出保存/不保存/取消

- **双层保存逻辑**
  - Session 层实时记录一切，重启不丢
  - 磁盘层只在用户显式 Cmd+S 时才写入原文件
  - 两层互不干扰

- **无格式粘贴**
  - 默认粘贴即纯文本，无需额外操作
  - 从网页、微信、PDF 复制进来的文字，粘贴自动去格式
  - 可当"格式清理中转站"：富文本粘贴进来 → 再复制出去 → 干净文本

- **Markdown 可选渲染**
  - 默认纯文本编辑，`Cmd+R` 切换渲染视图
  - 渲染视图下不可编辑，切回纯文本继续写
  - .mint / .txt / .md 三种文件类型在设置中有独立的渲染开关

- **Finder 集成**
  - 任意文件夹右键 → "新建 MiniNote 文档"，直接创建 .mint 文件
  - 支持 macOS 原生 Quick Look，选中文件按空格即可预览
  - .mint 文件默认关联 MiniNote 打开

- **其他特性**
  - 底部状态栏显示行号/列号、字符数、编码、换行符类型、渲染模式
  - 支持三种文件格式：.mint（默认）、.txt、.md，另存为即可互相转换
  - 主题切换：浅色 / 深色 / 跟随系统
  - 设置页面内可检查 GitHub 更新

## 快捷键

| 功能 | 快捷键 |
|------|--------|
| 新建 | `Cmd+N` |
| 打开 | `Cmd+O` |
| 保存 | `Cmd+S` |
| 另存为 | `Cmd+Shift+S` |
| 关闭标签页 | `Cmd+W` |
| 撤销 / 重做 | `Cmd+Z` / `Cmd+Shift+Z` |
| 查找 | `Cmd+F` |
| 查找替换 | `Cmd+Option+F` |
| Markdown 渲染切换 | `Cmd+R` |
| 偏好设置 | `Cmd+,` |

## 支持的格式

| 格式 | 说明 |
|------|------|
| .mint | MiniNote 专属格式，默认新建。纯文本 + 轻量状态信息（光标位置、渲染状态） |
| .txt | 标准纯文本，兼容其他编辑器 |
| .md | Markdown 格式 |

三种格式本质都是纯文本，互相转换就是改个后缀的事。

## 安装

**方式一：DMG 安装包**

1. 进入本仓库的 [Releases](../../releases) 页面，下载最新的 `MiniNote-[版本号].dmg`
2. 打开 DMG，将 MiniNote 拖入 Applications 文件夹
3. 首次启动时，macOS 可能提示"应用已损坏"或"无法验证开发者"——这是 Gatekeeper 对未付费签名应用的正常拦截，应用本身完好。在终端执行以下命令移除隔离标记：
   ```bash
   xattr -cr /Applications/MiniNote.app
   ```
   之后双击即可正常启动；或右键点击 → 打开 → 弹窗中再点"打开"。

**方式二：ZIP 压缩包**

1. 进入本仓库的 [Releases](../../releases) 页面，下载最新的 `MiniNote-[版本号].zip` 并解压
2. 将 `MiniNote.app` 移动至 Applications 文件夹
3. 在终端执行：
   ```bash
   xattr -cr /Applications/MiniNote.app
   ```

**方式三：自行编译（无需绕过签名）**

1. 克隆本仓库
2. 用 Xcode 打开 `MiniNote.xcodeproj`
3. 在 **Signing & Capabilities** 中选择你自己的开发者账号
4. `⌘R` 运行，Xcode 会自动签名

## 使用

- **新建标签页**：`Cmd+N` 创建暂存文档，直接开始写，无需命名或选择路径
- **打开文件**：`Cmd+O` 打开磁盘上的 .mint / .txt / .md 文件
- **保存**：`Cmd+S` 保存当前标签页到磁盘；暂存文档会弹出另存为对话框
- **另存为**：`Cmd+Shift+S` 可将当前文档另存为不同格式（.mint ↔ .txt ↔ .md）
- **切换渲染**：`Cmd+R` 在纯文本编辑和 Markdown 渲染视图之间切换
- **查找替换**：`Cmd+F` 查找，`Cmd+Option+F` 查找并替换
- **标签页排序**：拖拽标签页即可调整顺序
- **关闭标签页**：`Cmd+W`，有未保存改动时会弹出保存提示
- **检查更新**：偏好设置（`Cmd+,`）内点击"检查更新"

## 常见问题

**暂存文档存在哪里？**

`~/Library/Application Support/MiniNote/sessions/` 目录下。每个暂存文档一个文件，外加一个 `session.json` 记录标签页顺序和元信息。

**.mint 和 .txt 有什么区别？**

本质完全相同，都是纯文本。区别仅在于 .mint 会额外保存光标位置和渲染状态等轻量信息，.txt 是标准纯文本不写入额外信息。两者随时可通过另存为互相转换，内容不变。

**支持语法高亮吗？**

不支持。MiniNote 是纯文本编辑器，保持纯粹。Markdown 渲染基于系统 AttributedString，提供基础的标题、列表、粗体等渲染，不做代码语法高亮。

**和系统文本编辑 / CotEditor / BBEdit 有什么区别？**

MiniNote 的核心理念是：标签页持久化（重启不丢）+ 双层保存逻辑（暂存 vs 磁盘分离）。系统文本编辑不支持标签页持久化；CotEditor 功能更丰富但无此机制；BBEdit 太重。MiniNote 只做好这一件事。

**支持云同步吗？**

不支持，也不会支持。所有数据存在本地，完全离线。
