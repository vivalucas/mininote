# MiniNote

## Зачем я это сделал

Я перешёл с Windows на Mac и привык ко многому, но Блокнот Windows 11 мне всё время не хватал.

Не потому что он мощный — а потому что он простой. Открыл — пишешь, закрыл — ничего не потерял, вставил — и текст чистый. Без форматирования, без rich text, без «умных» подсказок. Просто тихое место для письма.

Я долго искал на macOS. Либо слишком тяжёлые (Obsidian, Typora), либо слишком простые (системный Текстовый редактор), либо по подписке. Ничего подходящего.

Поэтому я сделал свой.

## Возможности

- **Сохранение вкладок**
  - Создай вкладку и пиши, содержимое сохраняется в реальном времени
  - Выключение, перезагрузка, отключение питания — всё восстанавливается, ничего не теряется
  - При закрытии окна все вкладки тихо сохраняются, без уведомлений
  - При закрытии отдельной вкладки с несохранёнными изменениями: Сохранить / Не сохранять / Отмена

- **Двухуровневая логика сохранения**
  - Слой сессии: записывает всё в реальном времени, переживает перезагрузку
  - Слой диска: записывает в файл только при явном Cmd+S
  - Оба слоя работают независимо

- **Вставка как чистый текст**
  - Вставка по умолчанию — чистый текст, без лишних действий
  - Текст, скопированный с сайтов, из WeChat, PDF, автоматически очищается от форматирования
  - Работает как «станция очистки формата»: вставил rich text → скопировал обратно → чистый текст

- **Рендеринг Markdown (опционально)**
  - По умолчанию обычное редактирование текста, Cmd+R переключает в режим рендеринга
  - В режиме рендеринга редактирование недоступно, возврат в текстовый режим для продолжения
  - .mint / .txt / .md имеют независимые переключатели рендеринга в настройках

- **Интеграция с Finder**
  - Правый клик на любой папке в Finder — создание нового файла .mint
  - Нативная поддержка Quick Look в macOS — выберите файл и нажмите пробел для предпросмотра
  - Файлы .mint открываются в MiniNote по умолчанию

- **Прочее**
  - Статусная строка: строка/столбец, количество символов, кодировка, тип переноса, режим рендеринга
  - Три формата: .mint (по умолчанию), .txt, .md — конвертация через «Сохранить как»
  - Переключение темы: Светлая / Тёмная / Системная
  - Проверка обновлений GitHub из Настроек

## Горячие клавиши

| Функция | Сочетание |
|---------|----------|
| Новый | `Cmd+N` |
| Открыть | `Cmd+O` |
| Сохранить | `Cmd+S` |
| Сохранить как | `Cmd+Shift+S` |
| Закрыть вкладку | `Cmd+W` |
| Отменить / Повторить | `Cmd+Z` / `Cmd+Shift+Z` |
| Найти | `Cmd+F` |
| Найти и заменить | `Cmd+Option+F` |
| Переключить Markdown | `Cmd+R` |
| Настройки | `Cmd+,` |

## Поддерживаемые форматы

| Формат | Описание |
|--------|----------|
| .mint | Собственный формат MiniNote, по умолчанию для новых файлов. Чистый текст + лёгкая информация о состоянии (позиция курсора, состояние рендеринга) |
| .txt | Стандартный чистый текст, совместимый с другими редакторами |
| .md | Формат Markdown |

Все три по сути являются чистым текстом. Конвертация между ними — просто смена расширения.

## Установка

**Способ 1: DMG-установщик**

1. Перейдите на страницу [Releases](../../releases) и скачайте последний `MiniNote-[версия].dmg`
2. Откройте DMG и перетащите MiniNote в папку Applications
3. При первом запуске macOS может сообщить «приложение повреждено» или «не удаётся проверить разработчика» — это нормальное поведение Gatekeeper для неподписанных приложений. Выполните в Терминале:
   ```bash
   xattr -cr /Applications/MiniNote.app
   ```
   Затем дважды кликните для запуска. Или правый клик → Открыть → нажмите «Открыть» в диалоге.

**Способ 2: ZIP-архив**

1. Перейдите на страницу [Releases](../../releases) и скачайте последний `MiniNote-[версия].zip`, затем распакуйте
2. Переместите `MiniNote.app` в папку Applications
3. Выполните в Терминале:
   ```bash
   xattr -cr /Applications/MiniNote.app
   ```

**Способ 3: Сборка из исходного кода (без обхода подписи)**

1. Клонируйте этот репозиторий
2. Откройте `MiniNote.xcodeproj` в Xcode
3. В **Signing & Capabilities** выберите свой аккаунт разработчика
4. `Cmd+R` для запуска — Xcode автоматически подпишет приложение

## Использование

- **Новая вкладка**: `Cmd+N` создаёт черновой документ, сразу можно начинать писать
- **Открыть файл**: `Cmd+O` открывает .mint / .txt / .md файлы с диска
- **Сохранить**: `Cmd+S` сохраняет текущую вкладку на диск; для черновых документов открывается диалог «Сохранить как»
- **Сохранить как**: `Cmd+Shift+S` сохраняет в другом формате (.mint / .txt / .md)
- **Переключение рендеринга**: `Cmd+R` переключает между текстовым редактированием и режимом Markdown
- **Найти и заменить**: `Cmd+F` для поиска, `Cmd+Option+F` для поиска и замены
- **Изменение порядка вкладок**: перетащите вкладки для изменения порядка
- **Закрыть вкладку**: `Cmd+W`, запрашивает подтверждение при несохранённых изменениях
- **Проверка обновлений**: в Настройках (`Cmd+,`) есть кнопка «Проверить обновления»

## Часто задаваемые вопросы

**Где хранятся черновые документы?**

В директории `~/Library/Application Support/MiniNote/sessions/`. Каждый черновой документ — отдельный файл, плюс `session.json` с порядком вкладок и метаданными.

**В чём разница между .mint и .txt?**

По сути идентичны — оба чистый текст. Единственное отличие: .mint дополнительно сохраняет позицию курсора и состояние рендеринга. Свободно конвертируются через «Сохранить как» без потери содержимого.

**Поддерживается ли подсветка синтаксиса?**

Нет. MiniNote — текстовый редактор. Рендеринг Markdown использует системный AttributedString для базовых заголовков, списков, жирного текста и т.д. Подсветка синтаксиса кода не предусмотрена.

**Чем отличается от TextEdit / CotEditor / BBEdit?**

Ключевая философия MiniNote: сохранение вкладок (переживает перезагрузку) + двухуровневая логика сохранения (разделение черновиков и диска). TextEdit не поддерживает сохранение вкладок. CotEditor богаче функциями, но не имеет подобного механизма. BBEdit слишком тяжёлый. MiniNote делает одну вещь, но делает её хорошо.

**Поддерживается ли облачная синтронизация?**

Нет, и никогда не будет. Все данные хранятся локально, полностью офлайн.
