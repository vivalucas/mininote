# MiniNote

macOS 用の軽量テキストエディタ。Windows 11 のメモ帳の体験を目標にしています。

## なぜ作ったか

Windows から Mac に乗り換えて、ほとんどのことは慣れた。でもひとつだけ、Windows 11 のメモ帳がずっと恋しかった。

強力だからではない。むしろ、シンプルだからだ。開けば書けて、閉じても消えない、貼り付ければきれいなテキストになる。書式なし、リッチテキストなし、"スマート"な提案なし。ただ静かに文字が書ける場所。

macOS で長い間探した。重すぎるもの（Obsidian、Typora）、簡素すぎるもの（標準のテキストエディット）、サブスクが必要なもの。ちょうどいいものがなかった。

だから自分で作った。

## 機能

- **タブ永続化**
  - 新しいタブを作ればすぐ書ける。内容はリアルタイムで自動保存
  - シャットダウン、再起動、停電でも内容は完全復元。何も失われない
  - ウィンドウを閉じても、すべてのタブがサイレントに保持。プロンプトなし
  - 保存されていない変更があるタブを閉じると、保存 / 保存しない / キャンセルが表示

- **二層保存ロジック**
  - Session レイヤー：リアルタイムで全記録。再起動しても消えない
  - ディスクレイヤー：ユーザーが Cmd+S を押した時だけファイルに書き込み
  - 二つのレイヤーは独立して動作

- **プレーンテキスト貼り付け**
  - デフォルトで貼り付けはプレーンテキスト。追加操作不要
  - Webページ、WeChat、PDF からコピーしたテキストは自動的に書式が除去
  - 「書式クリーニングステーション」として使える：リッチテキストを貼って、コピーし直せばクリーンなテキスト

- **Markdown レンダリング（オプション）**
  - デフォルトはプレーンテキスト編集。Cmd+R でレンダリングビューに切り替え
  - レンダリングビューでは編集不可。プレーンテキストに戻れば編集再開
  - .mint / .txt / .md それぞれ設定でレンダリングのON/OFFを独立制御

- **Finder 連携**
  - Finder の任意のフォルダを右クリックで .mint ファイルを作成
  - macOS ネイティブ Quick Look 対応。ファイルを選んでスペースキーでプレビュー
  - .mint ファイルはデフォルトで MiniNote で開く

- **その他**
  - ステータスバーに行/列、文字数、エンコーディング、改行コード、レンダリングモードを表示
  - 3つのファイル形式：.mint（デフォルト）、.txt、.md。名前を付けて保存で相互変換
  - テーマ切替：ライト / ダーク / システムに従う
  - 設定画面から GitHub のアップデートをチェック

## 対応フォーマット

| フォーマット | 説明 |
|-------------|------|
| .mint | MiniNote 独自形式。新規作成のデフォルト。プレーンテキスト + 軽量状態情報（カーソル位置、レンダリング状態） |
| .txt | 標準プレーンテキスト。他のエディタとの互換性あり |
| .md | Markdown 形式 |

3つとも本質的にはプレーンテキスト。相互変換は拡張子を変えるだけ。

## インストール

**方法1：DMG インストーラー**

1. [Releases](../../releases) ページから最新の `MiniNote-[バージョン].dmg` をダウンロード
2. DMG を開き、MiniNote を Applications フォルダにドラッグ
3. 初回起動時、macOS が「アプリが破損しています」または「開発者を確認できません」と表示することがあります。これは未署名アプリに対する Gatekeeper の通常動作です。ターミナルで以下を実行して隔離フラグを解除：
   ```bash
   xattr -cr /Applications/MiniNote.app
   ```
   その後ダブルクリックで起動。または右クリック → 開く → ダイアログで「開く」をクリック。

**方法2：ZIP アーカイブ**

1. [Releases](../../releases) ページから最新の `MiniNote-[バージョン].zip` をダウンロードして解凍
2. `MiniNote.app` を Applications フォルダに移動
3. ターミナルで実行：
   ```bash
   xattr -cr /Applications/MiniNote.app
   ```

**方法3：ソースからビルド（署名ワークアラウンド不要）**

1. このリポジトリをクローン
2. Xcode で `MiniNote.xcodeproj` を開く
3. **Signing & Capabilities** で自分の開発者アカウントを選択
4. `Cmd+R` で実行。Xcode が署名を自動処理

## 使い方

- **新規タブ**：`Cmd+N` でスクラッチ文書を作成。すぐに書き始められる
- **ファイルを開く**：`Cmd+O` でディスク上の .mint / .txt / .md ファイルを開く
- **保存**：`Cmd+S` で現在のタブをディスクに保存。スクラッチ文書は名前を付けて保存ダイアログが表示
- **名前を付けて保存**：`Cmd+Shift+S` で別の形式として保存（.mint / .txt / .md）
- **レンダリング切替**：`Cmd+R` でプレーンテキスト編集と Markdown レンダリングビューを切り替え
- **検索と置換**：`Cmd+F` で検索、`Cmd+Option+F` で検索と置換
- **タブ並べ替え**：タブをドラッグで順序変更
- **タブを閉じる**：`Cmd+W`。未保存の変更があれば保存確認が表示
- **アップデート確認**：環境設定（`Cmd+,`）の「アップデートを確認」ボタン

## よくある質問

**スクラッチ文書はどこに保存されますか？**

`~/Library/Application Support/MiniNote/sessions/` ディレクトリ。各スクラッチ文書は個別ファイルとして保存され、`session.json` にタブ順序とメタ情報が記録されます。

**.mint と .txt の違いは？**

本質的に同一。どちらもプレーンテキスト。唯一の違いは、.mint がカーソル位置とレンダリング状態を追加保存すること。名前を付けて保存で相互変換可能で、内容は変わりません。

**シンタックスハイライトはサポートしていますか？**

いいえ。MiniNote はプレーンテキストエディタです。Markdown レンダリングはシステムのAttributedStringを使用し、基本的な見出し、リスト、太字などを表示します。コードのシンタックスハイライトは行いません。

**TextEdit / CotEditor / BBEdit とはどう違いますか？**

MiniNote の核となる理念は：タブ永続化（再起動で消えない）+ 二層保存ロジック（スクラッチとディスクの分離）。TextEdit はタブ永続化をサポートしていない。CotEditor はより多機能だがこの仕組みがない。BBEdit は重すぎる。MiniNote はこの一つだけを極める。

**クラウド同期はサポートしていますか？**

いいえ。今後もサポートしません。すべてのデータはローカルに保存され、完全にオフラインです。
