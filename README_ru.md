# MiniNote

MiniNote — локальное настольное приложение для заметок. Откройте его, сразу пишите, храните всё на своём компьютере и закрепляйте заметки на рабочем столе, когда они нужны. Без аккаунта, без облачной синхронизации и без тяжёлого процесса ведения базы знаний.

## Для чего подходит

- Быстро записывать идеи, команды, задачи, фрагменты встреч или подсказки для работы/игр.
- Держать часто используемый текст закреплённым на рабочем столе, чтобы быстро читать и копировать.
- Открывать небольшое окно заметки, не прерывая текущую работу.
- Организовывать локальные черновики простыми категориями.
- Писать лёгкий Markdown и быстро просматривать заголовки, списки, цитаты и блоки кода.

## Основные возможности

- **Локальная библиотека заметок**: заметки, категории и настройки хранятся локально; сначала выбирать путь к файлу не нужно.
- **Быстрые заметки**: небольшое окно можно открыть из системного трея или глобальным сочетанием клавиш; оно может появляться рядом с курсором.
- **Плитки на рабочем столе**: закрепляйте заметку на экране с пользовательскими цветами и опциональным рендерингом Markdown.
- **Предпросмотр Markdown**: подходит для повседневного структурированного текста, но не заменяет полноценную Markdown-IDE.
- **Импорт и экспорт**: импортируйте отдельный файл или папку как категорию; поддерживаются `.mint`, `.md`, `.markdown` и `.txt`.
- **Защита синхронизации с исходным файлом**: импортированные и экспортированные файлы могут сохранять ссылку на источник; перед записью обратно MiniNote проверяет внешние изменения.
- **Настраиваемый внешний вид**: тема, размер шрифта, фоновое изображение, горячие клавиши и закрытие в трей настраиваются.

## Поддерживаемые форматы

| Формат              | Назначение                                            |
| ------------------- | ----------------------------------------------------- |
| `.mint`             | Формат документа по умолчанию в MiniNote; текст UTF-8 |
| `.md` / `.markdown` | Документ Markdown                                     |
| `.txt`              | Обычный текстовый файл                                |

Все поддерживаемые файлы можно открыть в обычном текстовом редакторе. MiniNote не записывает приватные метаданные в тело файла.

## Установка и обновление

Официальные сборки публикуются на [GitHub Releases](https://github.com/vivalucas/mininote/releases). Начиная с версии 1.0.0 официальные файлы релиза предоставляются только для Windows и macOS. Поддержка Linux остаётся в исходном коде, но официальный пакет Linux пока не публикуется.

### Windows

- Установщик: `mininote-<version>-windows-x64-setup.exe`
- Портативная версия: `mininote-<version>-windows-x64.exe`

Для обычного использования выбирайте установщик. Портативная версия подходит для временного запуска или работы из фиксированной папки.

### macOS

Пользователям Apple Silicon нужно скачать `mininote-<version>-macos-arm64.dmg`, открыть его и перетащить `MiniNote.app` в `Applications`.

Сборка для macOS сейчас не подписана официально. Если macOS сообщает, что приложение нельзя открыть, оно повреждено или получено от неустановленного разработчика, сначала убедитесь, что файл скачан со страницы Release этого проекта, затем выполните:

```bash
xattr -cr /Applications/MiniNote.app
```

После этого откройте приложение снова. Для обновления закройте MiniNote, скачайте новый DMG и замените старое приложение в `Applications`.

### Linux

MiniNote 1.0.0 не поставляется с официальными пакетами Linux. Если нужна Linux-сборка, соберите её из исходного кода; конфигурация пакетирования Linux остаётся в репозитории.

## Где хранятся данные

MiniNote не загружает заметки и не предоставляет облачную синхронизацию. Заметки, настройки и индексные данные по умолчанию хранятся в папке `MiniNote` внутри системного каталога данных приложения. Если задана переменная `MININOTE_DATA_DIR`, MiniNote использует этот каталог.

## Сборка из исходного кода

Нужны Node.js, Rust и системные зависимости, требуемые Tauri.

```bash
npm ci
npm run tauri build
```

Режим разработки:

```bash
npm run tauri dev
```

## Границы

- Без аккаунтов и облачной синхронизации.
- Без редактора форматированного текста.
- Без полноценной Markdown-IDE.
- Без сложной базы знаний, обратных ссылок и совместной работы.

MiniNote должен оставаться лёгким, быстрым, локальным и удобным.
